SuperviseAI Architecture

Current implementation snapshot for the hackathon build (updated for the NestJS stack).

Frontend

React + Vite + TypeScript with a custom lightweight router and Socket.IO client.

Backend

NestJS + TypeORM + JWT auth + Socket.IO realtime gateway.

Data + Files

PostgreSQL for relational data and MinIO (S3-compatible) for thesis file storage.

AI + Integrations

Azure OpenAI, Azure Speech, Azure Language/Cognitive, Copyleaks, Semantic Scholar.

Runtime Topology

Browser app calls /api/v1/* on the NestJS API. API handles auth, role guards, uploads, AI orchestration, and persistence. Client subscribes to realtime events over Socket.IO using JWT handshake.

Client: React API: NestJS Data: Postgres + MinIO

Core Modules

Module Responsibility Notes
auth Register/login, password reset flow, JWT issuance, /auth/me. bcrypt + JWT strategy + role guard.
theses Student thesis proposal/workspace and professor dashboard data assembly. Includes trend, risk, history, and diff payload shaping.
submissions PDF/DOCX upload, text extraction, analysis/citation/plagiarism pipeline. Emits realtime progress events per stage.
coaching Argument defender/socratic/mock viva sessions. Adaptive difficulty + learner profiles + live turn metrics.
cohorts + milestones Professor cohorts, enrollments, milestone tracking and completion. Used by student and professor views.
realtime Socket.IO server with JWT auth and room fanout. Rooms: user:{id}, professor:{id}.
webhooks Copyleaks callback ingestion and plagiarism report finalization. Public endpoint for async scan completion.

Data Model (High Level)

Main entities: users, password_resets, theses, submissions, thesis_analysis, citation_reports, plagiarism_reports, coaching_sessions, cohorts, enrollments, milestones.

Implemented: migrations + TypeORM entities auto-loaded at startup. Note: external integrations are optional and have heuristic fallback behavior.

AI Pipeline and Fallbacks

  • Thesis analysis tries Azure OpenAI first; falls back to deterministic heuristics if unavailable.
  • Citation pipeline runs 3 layers: regex extraction, GPT format validation (if available), Semantic Scholar verification.
  • Plagiarism runs Copyleaks async scan when configured; otherwise sentence-repetition heuristic fallback.
  • Coaching uses adaptive loops with confidence/sentiment scoring and learner profile-aware prompts.

Environment Keys (Current)

Key groups used by the running app:

  • Core: PORT, DATABASE_URL, JWT_SECRET, FRONTEND_URL, CORS_ORIGIN, RESET_TOKEN_TTL_MINUTES, API_BASE_URL
  • MinIO: MINIO_*
  • Azure OpenAI: AZURE_OPENAI_*
  • Azure Speech: AZURE_SPEECH_*
  • Azure Language/Cognitive: AZURE_LANGUAGE_* or AZURE_COGNITIVE_* (both accepted)
  • Copyleaks: COPYLEAKS_EMAIL, COPYLEAKS_API_KEY
  • Semantic Scholar: SEMANTIC_SCHOLAR_API_KEY (optional)
